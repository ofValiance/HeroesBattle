# HeroesBattle

1) Метод generate интерфейса GeneratePreset
   - Возвращает объект армии со списком всех юнитов, отобранных по атаке в первую очередь и здоровью во вторую, набранных согласно ограниченям макс. цены армии и кол-ва юнитов одного типа.
   - Каждому юниту назначено имя в формате "Имя типа + номер среди своего типа". Юниты случайно распределены по доступным для армии клеткам.
   - Алгоритм реализован с помощью динамического программирования по двум параметрам (атака, здоровье) с оптимизацией в виде бинарного разложения максимального числа юнитов одного типа.
   - Сложность:
       - Внешний цикл dp = O(n)
       - Бинарное разложение кол-ва юнитов = O(log(k)) = O(1)
       - Внутренний цикл по стоимости = O(W)
       - Выбор лучшего dp = O(W)
       - Создание списка юнитов армии = O(n + m * k) = O(m)
       - Расстановка юнитов = O(m)
       - Итого = O(n) * O(1) * O(W) + O(W) + O(m) + O(m) = O(n * W)

2) Метод simulate интерфейса SimulateBattle
   - Организует последовательность атаки юнитов по раундам и ходам. Общий список юнитов сортируется по возрастанию атаки, разные армии в нем перемешаны.
   - Каждый ход список юнитов обновляется, чтобы исключить мертвых юнитов и пересортировать живых. Когда одна из армий остается без юнитов, цикл раундов заканчивается.
   - Алгоритм использует один основной цикл (раунды) и вложенные в него (этапы и проверки хода).
   - Сложность:
       - Цикл раундов = O(n)
       - Сбор всех живых юнитов = O(n)
       - Проверка на проигравшего = O(n)
       - Сортировка по возрастанию атаки = O(n * log(n))
       - Цикл атаки = O(n)
       - Итого = O(n) * (O(n) + O(n) + O(n * log(n)) + O(n)) = O(n^2 * log(n))

3) Метод getSuitableUnits интерфейса SuitableForAttackUnitsFinder
   - Находит юниты противника доступные для атаки, то есть не прикрытые слева (для армии компьютера) или справа (для армии игрока) и возвращает их список.
   - Алгоритм использует два цикла: основной для поиска по колонкам армии и вложенный для поиска по рядам армии.
   - Сложность:
       - Внешний цикл по колонкам = O(columns)
       - Внутренний цикл по рядам = O(rows)
       - Операции в циклах = O(1)
       - Итого = O(columns * rows)

4) Метод getTargetPath интерфейса UnitTargetPathFinder
   - Возвращает список координат клеток, которые нужно преодолеть атакующему юниту, чтобы дойти до цели. Движение возможно в 8 направлениях.
   - Метод находит клетки, занятые другими юнитами и недоступные к проходу, ищет доступную среди ближайших клеток цели и просчитывает путь, если такая клетка есть.
   - Для поиска пути используется алгоритм дейкстры с приоритетной очередью.
   - Сложность:
       - Построение карты препятствий = O(width * height + n) = O(N + U) = O(n)
       - Поиск ближайшей доступной клетки рядом с целью = O(1)
       - Алгоритм дейкстры с вспомогательным операциями = O(N) + O(N * log(N)) + O(N) = O(N * log(N))
       - Итого = O(N * log(N) + n)
